# 第一部分：项目宪章

## 1.1 项目本质

**这是一个法式古典家具出口企业的"AI辅助报价中枢系统"**，不是单纯的报价工具，而是从"客户询价"到"工厂生产存档"的全链路数字化平台。

## 1.2 三大核心能力

```
┌─────────────────────────────────────────────────────────┐
│                     核心能力                             │
├─────────────────────────────────────────────────────────┤
│  1. 看懂图片 → AI识别家具特征（雕花/材质/工艺）          │
│  2. 算对价格 → 五维定价矩阵 + 历史参考                   │
│  3. 出对方案 → 3D配置器 + 多语言报价 + 客户门户         │
└─────────────────────────────────────────────────────────┘
```

## 1.3 目标用户

| 角色 | 使用工具A | 使用工具B | 使用客户门户 |
|---|---|---|---|
| 内部标注员 | ✅ 主要使用者 | — | — |
| 业务员 | 偶尔查询 | ✅ 主要使用者 | — |
| 分公司总经理 | 偶尔查询 | ✅ 审核环节 | 审批客户注册 |
| 最终决策人 | — | 特殊报价审核 | — |
| 海外客户 | — | — | ✅ 主要使用者 |

## 1.4 三个不可妥协的原则

> 🔴 **原则一：定价权完全在公司**
> AI是辅助决策，不自动定价。最终价格永远由人决定。

> 🔴 **原则二：核心数据绝不外泄**
> 服务器+数据库部署在国内自己的服务器。外部访问只通过端口入口，运算和存储都在内部。

> 🔴 **原则三：客户看不到任何定价逻辑**
> 客户在配置器里自由玩，但看不到任何价格。报价由业务员单独发送。

## 模块 M-10：报价引擎（最核心模块）

### 模块定位
五维定价矩阵的计算引擎，提供"输入产品特征+客户信息→输出建议报价"的能力。

### 五维定价公式
```
建议报价 = 基准价 × L1 × L2 × L3 × L4 × L5

其中：
  基准价   = 历史相似产品的报价中位数
  L1 客户级别系数  = 1.0 / 1.2 / 1.5-1.6 / ≈1.0 / 锁价
  L2 数量阶梯系数  = 1件×1.0 / 3-10件×0.92 / 10-50×0.82 / 50-100×0.75 / 100+×0.68
  L3 汇率系数      = 实时汇率 × 是否锁价
  L4 客户承价系数  = A×1.1-1.2 / B×1.0 / C×0.9-0.95
  L5 地区消费系数  = US×1.3 / AU×1.2 / ME×1.3 / EE×1.1 / SEA×1.0 / CN×1.0
```

### 关键业务规则

| 规则 | 说明 |
|---|---|
| **AI不自动定价** | 引擎只输出"建议价格"，业务员手动确认或调整 |
| **历史参考优先** | 优先调用RAG检索类似产品历史报价，基准价基于历史数据 |
| **L5客户锁价豁免** | 美国批发商等L5客户，价格不随汇率自动调整，需人工决策 |
| **汇率波动预警** | >3%自动通知，提示是否调价 |
| **承价系数双层** | 客户档案设默认值（A/B/C），业务员每次询价可微调 |
| **完全对客户隐藏** | 所有系数仅业务员可见，客户只看最终报价 |

### 输入 Schema

```python
class PricingRequest(BaseModel):
    product_features: dict   # AI识别+人工校正的产品特征
    customer_id: str         # 客户编号
    quantity: int            # 数量
    custom_acceptance: Optional[str] = None  # 临时调整承价等级 A/B/C
    note: Optional[str] = None
```

### 输出 Schema

```python
class PricingResponse(BaseModel):
    suggested_price_cny: float       # 建议人民币基准价
    suggested_price_usd: float       # 美元价
    suggested_price_idr: float       # 印尼盾价
    breakdown: PricingBreakdown      # 五维系数明细（仅业务员可见）
    historical_references: List[Quote]  # RAG检索的相似历史报价
    confidence: str                  # high/medium/low（基于历史数据数量）
    warnings: List[str]              # 警告（如汇率波动、客户偏离基准）
```

### 依赖
- M-15 客户档案（取客户级别和承价等级）
- M-08 历史报价（RAG检索）
- M-19 汇率服务（实时汇率）

### 优先级
🔴 **P0** — 工具B的核心，必须最先开发

## 模块 M-12：材质效果图生成（Seedream集成）

### 模块定位
基于3D模型，根据客户选择的材质/油漆/面料，生成高清效果图。

### 关键业务规则

| 规则 | 说明 |
|---|---|
| **3角度同时出图** | 正视图+侧视图+侧后方视图，每次换材质=3次API调用 |
| **支持多方案对比** | 同一产品并排展示不同材质方案 |
| **材质库受限** | 客户只能选预设材质（=工厂能力上限），不能自由输入材质 |
| **允许参考图上传** | 客户可上传参考图，系统标注"需工厂确认可行性" |
| **3-5秒等待可接受** | 不追求实时，质量优先 |

### 输入 Schema

```python
class MaterialRenderRequest(BaseModel):
    glb_model_path: str           # 3D模型路径
    material_config: dict          # {部位: 材质ID}
    angles: List[str] = ["front", "side", "side_back"]
    custom_reference: Optional[str] = None  # 客户上传的参考图
```

### 输出 Schema

```python
class MaterialRenderResult(BaseModel):
    rendered_images: dict          # {angle: image_url}
    config_id: str                 # 该配置的唯一编号（用于报价单引用）
    cost_usd: float                # 本次API成本
    duration_ms: int
```

### 依赖
- M-11 3D建模（产生GLB模型）
- M-13 材质库（材质ID查询）
- Seedream 4.0 Edit API

### 优先级
🔴 **P0** — 与M-11配套开发

## 模块 M-16：业务流程引擎

### 模块定位
管理14步业务流程的状态机。

### 状态流转

```
[INQUIRY_RECEIVED] 收到询价
        ↓
[QUOTE_DRAFTING] 业务员制作报价
        ↓
[QUOTE_REVIEWING] 总经理审核
        ↓ (审核通过)
[QUOTE_SENT] 发送给客户
        ↓
[NEGOTIATING] 还价中（可能多轮）
        ↓
[QUOTE_CONFIRMED] 价格确认
        ↓
[DEPOSIT_PAID] 收到50%定金 ★ 关键节点
        ↓ (触发出图)
[DRAWING_TYPE_A] CAD外形图生成中
        ↓
```

## 模块 M-19：安全防泄露

### 模块定位
横切多个模块的安全保护层。

### 防护策略

| 威胁 | 防护手段 |
|---|---|
| 客户截图泄露报价 | 报价单唯一编号+不可见水印（含客户ID+时间戳） |
| 同行冒充客户注册 | 人工审批 + 设备指纹 + IP监控 |
| 业务员账号被盗 | 钉钉强绑定 + 异常行为预警 |
| 客户账号被盗 | 短信OTP + 设备绑定 |
| 数据外泄 | 服务器内网部署 + 操作审计日志 |
| 内部人员泄露 | 操作权限分级 + 全量日志审计 |

### 远期升级
> 🔴 **"甚至于注册苹果的开发者，发布自己的App给客户专门使用。"**

独立 iOS/Android App + 设备硬件锁定 + 短信即时密码

### 优先级
🟠 **P1**（基础水印和审批）+ 🔵 **P2**（设备绑定和App）
