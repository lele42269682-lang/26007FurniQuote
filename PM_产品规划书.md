## 第一章 产品定位与战略价值

### 1.1 产品愿景（Vision）

**让法式古典家具的"图片询价 → 客户付定金"周期从 2-3 天压缩到 24 小时以内。**

### 1.2 用户与场景

| 用户 | 场景 | 痛点 | 我们的方案 |
|---|---|---|---|
| 业务员 | 收到客户随手发的家具图，需出报价 | 手工查历史报价、估算系数、做效果图，至少半天 | AI 识别 + 五维系数自动计算 + 历史 RAG 检索，5 分钟出报价草稿 |
| 总经理 | 审核业务员报价，担心定价混乱 | 不同业务员对同一客户报价差异大 | 报价单经审核流，五维系数全程留痕，差异自动告警 |
| 海外客户 | 想在不同方案间比较 + 旋转查看 | 一来一回邮件 / WhatsApp 沟通耗时 | 客户门户自助配置（材质/尺寸/面料），3D 旋转查看 |
| 工厂厂长 | 接到订单后要拆单出 CAD | 业务员发的报价单没有工程信息 | 报价 → 业务流 → CAD 工程图全链路绑定产品编号 |
| 标注员（工具A） | 给历史报价图打标签训练模型 | 已上线（74 测试通过） | 工具A → 工具B 数据互通 |

### 1.3 战略价值（为什么值得做）

> 引用需求方原话："**同行主要还是手工报价**"（Q21）

| 价值维度 | 说明 |
|---|---|
| **效率壁垒** | 24h vs 行业 2-3 天，订单转化率显著提升 |
| **数据壁垒** | 100+ 历史 Excel + 200 个 CNC STL/STEP 文件 → 训练数据 + 3D 资产，2-3 年内同行无法复刻 |
| **客户黏性** | 海外客户在客户门户自助配置已是国内同行未触及的服务标准 |
| **定价护城河** | 五维系数模型沉淀了多年定价经验，模型一旦稳定即成核心 IP |
| **可复制性** | 法式古典验证后，可平移至中式 / 现代等品类 |

### 1.4 不做什么（产品边界）

- ❌ **不做**通用家具 ERP（市面已有，差异化为 0）
- ❌ **不做**生产管理系统（拆单/排产留给现有工厂系统）
- ❌ **不做**电商交易（无 C 端支付能力）
- ❌ **不做**对外开放 API（核心数据国内私有部署）

## 第四章 阶段 1 详细规划：3D 配置器小模块（重点关注）

### 4.1 阶段目标

业务员收到客户图后：
1. 上传图片 → 系统生成 3D 模型（混元 3D）
2. 系统按预设材质渲染高清效果图（Seedream）
3. 在浏览器里旋转查看 + 切换材质（Three.js）
4. 把 URL 链接发给客户，客户自助选择
5. 客户提交后生成"配置方案"，业务员单独发送报价

**不包含**：客户门户登录（阶段 4）/ 报价单生成（阶段 2）/ 钉钉审批（阶段 3）

### 4.2 验收标准（DoD · Definition of Done）

| 验收项 | 通过标准 |
|---|---|
| 客户图片 → GLB 成功率 | ≥ 80%（10 张测试图片） |
| 客户专有产品保护 | `is_proprietary=true` 的图片**不调用云 API**（单元测试覆盖） |
| 材质换图速度（高清） | ≤ 5 秒/组 |
| 材质换图速度（预览） | ≤ 200ms（Three.js 实时贴图） |
| 桌面端旋转流畅度 | ≥ 50 FPS |
| 移动端旋转流畅度 | ≥ 30 FPS（iPhone 12 标准） |
| 三视图截图 | 正视图/侧视图/侧后方视图自动生成 |
| 多方案对比 | 同一产品同时配置 ≥ 3 个方案 |
| 自定义尺寸 | 推荐尺寸 + 自定义输入框双轨 |

### 4.3 关键风险与对策

| 风险 | 概率 | 影响 | 对策 |
|---|---|---|---|
| 混元 3D 对法式雕花识别精度不足 | 中 | 高 | 雕花部分使用现有 200 个 CNC STL/STEP 文件直接渲染，跳过 AI 生成 |
| Seedream 风格与客户期望不符 | 中 | 中 | 第一版限定 5 种材质模板，业务员人工把关 |
| Three.js 在低端手机上卡顿 | 中 | 中 | LOD（多分辨率）+ 离线打包贴图 |
| 客户专有产品误上云 | 低 | **极高（法律风险）** | 双重检查：模块入口 + 网关入口；单元测试覆盖率 100% |

### 4.4 阶段 1 验收 Demo 脚本

**演示场景**：业务员 Leslie 收到中东客户发来的法式扶手椅图片，需要给出配置链接。

```
1. 业务员登录后台（暂用占位登录）
2. 在 Dashboard 点击"上传客户图片"
3. 系统调用 m11 → 30 秒后生成 GLB
4. 业务员选择 5 种预设材质（樱桃木+真金箔 / 胡桃木+亚光金 / ...）
5. 系统调用 m12 → 各 5 秒生成高清渲染图
6. 业务员复制配置器链接发给客户
7. 客户在浏览器里：
   - 旋转 3D 模型（流畅）
   - 切换材质（毫秒级）
   - 输入自定义尺寸（宽 700mm × 深 750mm × 高 950mm）
   - 提交方案
8. 业务员后台收到方案 → 进入阶段 2 报价流程
```

4. 业主认定五维系数模型已被业务员/外包带走（IP 失守）
5. 海外客户在测试期反馈门户体验不如手工沟通（产品方向错误）
